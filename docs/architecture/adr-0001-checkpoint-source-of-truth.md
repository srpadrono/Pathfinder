# ADR-0001: Checkpoint Source of Truth

- **Status:** Accepted
- **Date:** 2026-02-12
- **Deciders:** Pathfinder maintainers

## Context

Pathfinder previously had two overlapping checkpoint data paths:

1. Reporter-derived checkpoint results (`test-results/checkpoints.json`) from test titles.
2. Fixture-emitted marker artifact files (`test-results/markers-*.json`).

This created ambiguity and maintenance overhead.

## Decision

`test-results/checkpoints.json` generated by `e2e/reporters/pathfinder-reporter.ts`
is the **single source of truth** for checkpoint state consumed by downstream tooling.

Fixture marker artifact files are deprecated and removed from runtime output.

## Consequences

### Positive
- One canonical schema for coverage and map generation.
- Less artifact noise in `test-results/`.
- Simpler debugging and fewer drift paths.

### Trade-offs
- Rich per-test marker timelines are no longer emitted by default.
- If needed later, marker timelines can be reintroduced behind an explicit opt-in flag.
